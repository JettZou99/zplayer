- `ui/translation_popup.py` — 无边框弹窗：可点击字幕原文 + 翻译结果展示

**修改文件**：
- `ui/subtitle_list.py` — 新增 `word_double_clicked` 信号，连接 `itemDoubleClicked`
- `ui/main_window.py` — 集成弹窗 + Worker 生命周期 + 信号连接

**有道 API**：`https://dict.youdao.com/jsonapi?q={word}`，免费无需 API Key，国内可直连
- 音标路径：`ec.word[0].usphone` / `ec.word[0].ukphone` 或 `simple.word[0].usphone`
- 释义路径：`individual.trs`（结构化）或 `ec.word[0].trs[].tr[].l.i[].#text`

**Bug 修复（2026-07-07）**：
- 问题：点击单词提示 `QTextBrowser: No document for word:xxx`，无法翻译
- 根因：`_ClickableTextBrowser` 定义了自定义信号 `anchor_clicked`（下划线）但从未发射，而 QTextBrowser 内置 `anchorClicked`（驼峰）才是实际触发的信号。连接的是从未发射的自定义信号。
- 修复：移除自定义信号，连接内置 `anchorClicked(QUrl)` 信号，在 `_on_anchor_clicked` 中从 QUrl 提取单词。同时修复 `setSource` 签名以消除导航警告。

## 修复 _on_subtitle_row_selected 类型错误（2026-07-07）

**Bug**: 点击字幕行时抛出 `TypeError: '<=' not supported between instances of 'int' and 'str'`

**根因**: `main_window.py` 中 `word_double_clicked` 信号（传递 `str, int`）被错误连接到 `_on_subtitle_row_selected`（只接受 `int`）。双击字幕时信号传递字符串作为 index 参数，导致 `0 <= index` 比较失败。

**修复**:
1. 删除 `main_window.py` 中错误的信号连接：`word_double_clicked.connect(_on_subtitle_row_selected)`
2. 清理 `subtitle_list.py`：删除不再使用的 `word_double_clicked` 信号和 `_handle_item_double_clicked` 方法

**文件变更**:
- `ui/main_window.py` — 删除错误连接
- `ui/subtitle_list.py` — 清理未使用信号和方法

## 添加词形还原功能（2026-07-07）

**需求**: 点击变形词（如 supports, testing, walked）时，自动识别为原型（support, test, walk）进行翻译。

**实现** (`ui/translator.py`):
1. 新增 `lemmatize(word)` 函数：将英文单词的词形变化还原为原型
2. 新增 `_IRREGULAR_FORMS` 字典：不规则动词/形容词/名词的原型映射
3. 修改 `normalize_query_word()`：调用 `lemmatize()` 进行词形还原
4. 修改 `YoudaoTranslator.translate()`：使用原型词查询API（缓存更高效）

**处理规则** (`lemmatize`):
1. 不规则词形映射表（better→good, children→child等）
2. `-ies` → `-y`（countries → country）
3. `-ied` → `-y`（studied → study）
4. `-es`（watches → watch, boxes → box）
5. `-s`（supports → support，排除特殊词this/us等）
6. `-ing`（running → run, making → make）
7. `-ed`（walked → walk, loved → love）
8. `-er`（faster → fast, nicer → nice）
9. `-est`（fastest → fast, nicest → nice）

**修复** (`ui/__init__.py`):
- 改为延迟导入（`__getattr__`），避免在非GUI环境下导入PyQt6时报错
- 这样可以在测试脚本中直接 `from ui.translator import lemmatize`

**测试结果**: 20个测试用例全部通过。

**文件变更**:
- `ui/translator.py` — 新增 `lemmatize()` + `_IRREGULAR_FORMS` + 修改 `normalize_query_word()`
- `ui/__init__.py` — 延迟导入MainWindow
- `ui/translation_panel.py` — 保存/恢复字幕原文HTML（前述修复）

## 单词收藏功能（2026-07-07）

为 ZPlayer 添加单词收藏/单词本功能，收藏翻译后的单词到 SQLite 数据库，关联字幕句子，支持搜索、删除和 CSV 导出。

**新增文件**：
- `data/__init__.py` — 数据持久化子包
- `data/models.py` — `CollectedWord` dataclass（word, definitions, phonetics, subtitle_text, subtitle_start/end_ms, video_name, collected_at, id）
  - `from_translation()` 从 TranslationResult 创建
  - `from_db_row()` / `to_db_dict()` 数据库序列化
  - `definitions_text` / `subtitle_time_range` 属性
- `data/wordbook_db.py` — `WordBookDB` SQLite 数据库层
  - 路径: `~/.zplayer/wordbook.db`
  - WAL 模式 + threading.Lock 线程安全
  - `add()` 使用 INSERT OR REPLACE（同词新例句自动更新）
  - `is_collected()`, `get_all()`, `search()`, `remove()`, `remove_by_id()`, `count()`, `export_csv()`
  - CSV 导出含 UTF-8 BOM（Excel 兼容）
- `ui/wordbook_dialog.py` — `WordBookDialog` 非模态对话框
  - QTableWidget 表格：单词/音标/释义/字幕句子/视频名/收藏时间
  - 实时搜索过滤、批量删除、CSV 导出、双击查看详情

**修改文件**：
- `ui/translation_panel.py` — 新增 `collection_toggled` 信号 + 收藏按钮(☆/★)
  - `set_subtitle_context()` 存储字幕上下文
  - `set_collection_state()` 切换收藏按钮状态
  - `_on_collect_clicked()` 发射 CollectedWord 或 str
  - `show_translation()` 中存储 `_current_result` 并显示收藏按钮
  - `show_loading()` 中隐藏收藏按钮
  - `clear_translation()` 重置收藏按钮
- `ui/main_window.py` — 集成收藏逻辑
  - 初始化 `WordBookDB`
  - 工具栏新增"单词收藏"按钮
  - 连接 `collection_toggled` → `_on_collection_toggled()`
  - `_on_subtitle_row_selected()` 传递字幕上下文
  - `_on_translation_success_panel()` 查询收藏状态
  - `_open_wordbook_dialog()` 非模态打开对话框
  - `closeEvent()` 关闭数据库

**数据流**：
```
点击字幕行 → set_subtitle_context() 存储上下文
点击单词翻译 → show_translation() + is_collected() 检查状态
点击☆收藏 → collection_toggled(CollectedWord) → DB.add() → ★已收藏
点击★取消 → collection_toggled(str) → DB.remove() → ☆未收藏
工具栏"单词收藏" → WordBookDialog → 搜索/删除/导出CSV
```
