# 2026-07-06 工作日志

## 修复视频加载失败 "无法创建媒体对象" 错误

**问题**: 用户打开 MP4 文件后报错 "无法创建媒体对象: **.mp4"

**根因分析**:
1. `vlc_setup.py` 中 `configure_vlc()` 只设置了 `PYTHON_VLC_MODULE_PATH`（python-vlc 内部使用），未设置 `VLC_PLUGIN_PATH`（libVLC 自身读取的环境变量）。当 libvlc.dll 通过 `os.add_dll_directory()` 加载时，可能无法自动定位插件目录，导致编解码器模块缺失。
2. `vlc_player.py` 向 `vlc.Instance()` 传入了 `--plugin-path=...` 参数，但该选项在 VLC 3.0 中已废弃（输出警告 "option --plugin-path no longer exists"），被静默忽略。
3. `_create_media()` 方法中正斜杠路径尝试（`media_new_path` 传入正斜杠路径）在 Windows 上总是返回 None，是无效尝试。
4. 错误日志级别为 DEBUG，难以排查。

**修复**:
- `player/vlc_setup.py`: 在 `configure_vlc()` 的两个路径（env var 已设 / 探测路径）都显式设置 `VLC_PLUGIN_PATH` 环境变量
- `player/vlc_player.py`: 移除废弃的 `--plugin-path` 实例参数；移除无效的正斜杠路径尝试；提升错误日志到 WARNING 级别；错误信息中包含各尝试方式的详细失败原因

**测试**: 使用系统 Python (C:\Users\zouxd\AppData\Local\Programs\Python\Python313) 验证，MP4、中文文件名、带空格文件名均加载成功。完整 PyQt6 应用流程测试通过。

**环境说明**: 项目 `.venv` 未安装依赖包，用户使用系统 Python 运行。VLC 3.0.23 安装在 `C:\Program Files\VideoLAN\VLC`。
